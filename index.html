Sobre o desafio

Para elaborar o desafio proposto, decidi manter as coisas relativamente simples. Como o projeto possui apenas uma página, evitei utilizar quaisquer ferramentas e bibliotecas JavaScript que pudessem aumentar a complexidade e o tamanho da aplicação, como o React. Como resultado, o site é composto apenas dos arquvios necessários: o index.html dá a estrutura da página; style.css, para estilização e criação dos layouts; e por fim um arquivo JavaScript index.js, para validar o formulário e buscar as informações do CEP.

Sem dúvidas, uma aplicação sem frameworks pode não ser adequada para um projeto de grande escala (para ver projetos mais complexos, visite meu site), mas para um projeto pequeno como este, a simplicidade é uma vantagem. O código pode ser repetitivo, mas evitar o uso de dependências força o desenvolvedor a criar soluções específicas para os problemas enfrentados, gerando um maior entendimento de como todas as peças se encaixarm. No final das contas, os usuários só precisam de uma aplicação que atenda suas necessidades.

  • Menos complexidade
  • Menos dependências
  • Menor custo
  • Mais controle
  • Mais rápido
  • Experiência do usuário
